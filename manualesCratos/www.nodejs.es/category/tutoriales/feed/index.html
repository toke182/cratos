(More)…]]> Git se ha transformado en uno de los sistemas de control de versiones más utilizados (por no decir el más utilizado) y por su lado GitHub en el servidor web con soporte git por excelencia. Es impresionante la cantidad de proyectos que se suman día a día y como de alguna manera esto ha modificado el paradigma de trabajo en el desarrollo de código para distintos proyectos.

Volviendo a git @tjholowaychuk ha creado hace un tiempo git-extras:

GIT utilities — repo summary, repl, changelog population, author commit percentages and more

Quiero simplemente comentar que no se trata solamente de git sino que también agrega muchas utilidades de github, por ejemplo “git issues“:

El uso de git-extras nos ayuda a hacer un mejor uso de buenas prácticas relacionadas con el sistema de control de nuestros proyectos como es el caso de “git changelog” y “git release“; comandos altamente recomendados.
Es importante practicar un poco en un repositorio de prueba antes de utilizarlos seriamente ya que podemos llegar a generar tags indeseados o registros incorrectos.
